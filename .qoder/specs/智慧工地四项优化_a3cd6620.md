# 智慧工地前端四项优化实施方案

## 概述

针对四项需求进行配置驱动式改造。核心原则：所有新参数集中在 `src/config/Config.js`，运行时行为通过各管理器新增单一公开方法实现，绝不重写既有函数或破坏 API 契约。

## 坐标系与布局约定

- 坐标系：中心为原点，X 轴向东，Z 轴向南，Y 轴向上
- 场地尺寸 `scene.size = 200`（边缘 ±100）
- 围墙置于场地边缘 ±100，高度 4
- 大门在南侧（z = +100）正中，宽度 16

## PDF 布置方案结论

经提取分析 `downloads/智慧工地布置方案.pdf`（1:200，"材料学院教学楼三维图"）：
- 该 PDF 为学生 3D 轴测布置图，**无围墙/大门/监控的明确文字标注**
- 布局推断：南侧为入口区域、北侧为塔吊区、西侧为生活办公区、东侧为加工作业区
- 塔吊位于场地北部、主教学楼东侧，臂架向西延伸
- 围墙范围、大门位置、监控点位基于场地规模和布局关系推断，在 3D 视图中微调校准

## 关键根因分析

| 需求 | 根因 | 修复策略 |
|------|------|----------|
| 对比度过大 | 缩远进雾区(fogNear=150)变暗 + 缩近 bloom(threshold=0.15)过曝 + OrbitControls 无距离限制 | 距离限制 + 白天主题雾更淡 + bloom threshold 提高 + exposure clamp 兜底 |
| 无昼夜切换 | 纯夜间深蓝主题，ambient/hemi 未存引用，无切换逻辑 | 补存引用 + setTheme 方法 + ThemeSwitch 组件 + 默认白天 |
| 地面穿帮 | 200×200 有限平面，转动视角看到边缘外虚空 | 围墙遮挡 + 地面扩展/裙边 + maxPolarAngle 已设 |
| 切换过快 | 指数 lerp 速度与距离成正比，无上限 | 以 Cam1→Cam3 距离÷参考时长推导速度上限，clamp 每帧位移 |

## Task 2: SceneManager 主题系统与曝光控制（需求1 + 需求2）

**文件**: `src/scene/SceneManager.js`
**依赖**: Task 1

### 2.1 补存光照引用

在 `_setupLights()`（约 L89-130）中，ambient 和 hemi 创建后补存为实例字段：
- `this._ambient = ambient;`（约 L91 后）
- `this._hemi = hemi;`（约 L98 后）
- `this._sun` 已存在（L129），无需改

### 2.2 新增 `_clampRender()` 方法

私有方法，用 `THREE.MathUtils.clamp` 把 `toneMappingExposure`、`bloomPass.strength` 限定在 Config 的 min/max 内。在构造函数末尾调用一次，在 `setTheme` / `updateExposure` 后也调用。

### 2.3 新增 `setTheme(themeName)` 方法

公开方法，读取 `CONFIG.theme[themeName]`，平滑过渡更新：
- 存储 `_targetTheme = CONFIG.theme[themeName]` 和 `_themeBlend = 0`（0→1 过渡进度）
- 在 `update(delta)` 中，当 `_themeBlend < 1` 时，对以下参数做 lerp 插值：
  - `scene.background`（THREE.Color.lerp）
  - `scene.fog.color/near/far`（Color.lerp + 线性插值）
  - `this._ambient.color/intensity`
  - `this._hemi.color/groundColor/intensity`
  - `this._sun.color/intensity/position`
  - `this._renderer.toneMappingExposure`
  - `this._bloomPass.strength/threshold`
- 进度因子用 `1 - Math.exp(-delta * rate)` 保证帧率无关，rate 按 `transitionDuration` 标定
- 过渡完成后调 `_clampRender()` 兜底
- 颜色 lerp 在 sRGB 空间插值，过渡时长 1.2s 足够短以避免中间灰暗色

### 2.4 构造函数默认白天

在构造函数末尾（光照、后处理都初始化完成后）调用 `this.setTheme(CONFIG.theme.default)`（即 `'day'`）。

### 2.5 OrbitControls 距离限制

在 OrbitControls 初始化处（约 L66-76）新增：
- `this._controls.minDistance = CONFIG.camera.minDistance;`
- `this._controls.maxDistance = CONFIG.camera.maxDistance;`

## Task 4: CameraManager 视角切换速度上限（需求4）

**文件**: `src/managers/CameraManager.js`
**依赖**: Task 1

### 4.1 init() 末尾动态计算参考速度

在 `init()`（约 L80-100）末尾，读取 cam1 和 cam3 的配置，计算：
```
const cam1 = CONFIG.cameras.find(c => c.id === 1);
const cam3 = CONFIG.cameras.find(c => c.id === 3);
// 飞行目标 = pos + flyOffset
const fly1 = [cam1.pos[0]+offset[0], cam1.pos[1]+offset[1], cam1.pos[2]+offset[2]];
const fly3 = [cam3.pos[0]+offset[0], cam3.pos[1]+offset[1], cam3.pos[2]+offset[2]];
const dist = Math.sqrt((fly3[0]-fly1[0])**2 + (fly3[1]-fly1[1])**2 + (fly3[2]-fly1[2])**2);
this._maxLinSpeed = dist / (CONFIG.cameraFly.refDurationMs / 1000);

const targetDist = Math.sqrt(
    (cam3.lookAt[0]-cam1.lookAt[0])**2 +
    (cam3.lookAt[1]-cam1.lookAt[1])**2 +
    (cam3.lookAt[2]-cam1.lookAt[2])**2
);
this._maxTargetSpeed = targetDist / (CONFIG.cameraFly.refDurationMs / 1000);
```
> cam1/cam3 缺失时回退安全默认值（maxLinSpeed=200, maxTargetSpeed=50）。

### 4.2 重构 update() 飞行逻辑（约 L221-234）

将现有纯指数 lerp 改为"先 lerp 方向再 clamp 位移"：
```
const k = 1 - Math.exp(-dt * CONFIG.cameraFly.lerpRate);

// 位置：期望位移 → 速度上限钳制
const desiredMove = this._flyToPos.clone().sub(this.camera.position).multiplyScalar(k);
const maxMove = this._maxLinSpeed * dt;
if (desiredMove.length() > maxMove) desiredMove.setLength(maxMove);
this.camera.position.add(desiredMove);

// 目标点：同理
const desiredTargetMove = this._flyToTarget.clone().sub(this.controls.target).multiplyScalar(k);
const maxTargetMove = this._maxTargetSpeed * dt;
if (desiredTargetMove.length() > maxTargetMove) desiredTargetMove.setLength(maxTargetMove);
this.controls.target.add(desiredTargetMove);

// 到达判定
if (this.camera.position.distanceTo(this._flyToPos) < CONFIG.cameraFly.arrivalThreshold &&
    this.controls.target.distanceTo(this._flyToTarget) < CONFIG.cameraFly.arrivalThreshold) {
    this._flying = false;
    this._controls.enabled = true;
}
```

### 4.3 提取硬编码偏移

将 L141 的 `pos.x+18, pos.y+12, pos.z+22` 改为读取 `CONFIG.cameraFly.flyOffset`。

> 该限速对 `selectCamera`、`restoreView`、再次点击恢复三条飞行路径统一生效，无需各自改动。CameraManager 报警链路（triggerAlert → _applyFlash → _restoreCamera）不受影响。

## 风险与缓解

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 主题切换颜色 lerp 经过中间灰暗色 | 中 | 过渡时长 1.2s 足够短；颜色用 THREE.Color.lerp 在 sRGB 空间插值 |
| 速度上限使长距离切换偏慢 | 中 | refDurationMs 可调（默认 1200ms）；短距离切换不受限（位移 < maxMove 时不钳制） |
| 围墙坐标与 PDF 布局有偏差 | 中 | PDF 无明确标注，坐标基于场地规模推断；围墙用分段配置便于微调；在 3D 视图中校准 |
| 入口监控视锥与围墙冲突 | 低 | FOV 增大后锥体变宽，depthWrite 已为 false 不遮挡；监控 pos.y=38 远高于围墙(4) |
| 改摄像头 FOV 破坏报警闪烁 | 低 | createCameraModel 结构不动，仅改 CONFIG 值；_findParts 按 name 查找不受影响；改后回归测试 4 路 triggerAlert |
| 白天 bloom 仍过亮 | 低 | 白天 bloomStrength=0.3 + threshold=0.6，仅保留高亮金属反光泛光 |
| 动态主题与曝光 clamp 冲突 | 低 | setTheme 写入后立即调 _clampRender 兜底；clamp 值覆盖主题写入值 |

## 被否决的方案

| 方案 | 来源 | 否决理由 |
|------|------|----------|
| 独立 EnvironmentManager.js 文件 | 方案B | 过度封装，增加文件数；setTheme 方法放 SceneManager 内更简洁，符合"克制设计"原则 |
| 动态曝光随距离映射 | 方案A | 与 setTheme 写同一字段冲突；需求根因是雾+bloom 而非 exposure 动态变化；改用距离限制 + 主题雾参数 + clamp 兜底更直接 |
| shadowMapSize 4096→2048 + 像素比钳制 + 移除 FXAA | 方案B | 用户未要求性能优化，属额外范围；降阴影分辨率可能引入视觉回归；新增围墙几何量小，不值得牺牲画质 |
| 围墙立柱用 InstancedMesh | 方案B | 仅 4 段墙体，InstancedMesh 增加复杂度无收益；BoxGeometry 更简洁可读 |
| 新增 2 路监控（西区/东区） | PDF agent | 用户要求"修正所有监控"非"新增"；保持 4 路摄像头不变 |
| 速度上限取固定 760 单位/秒 | 方案A | 等于当前无限制时的瞬时峰值，不产生实际限制效果；改用动态计算（距离÷参考时长）使上限有实际约束力 |
